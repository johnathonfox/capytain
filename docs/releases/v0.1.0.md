# QSL v0.1.0

> **Status:** feature-complete, not yet tagged. Three release blockers
> remain (see [Known limitations](#known-limitations)). Linux is the
> primary platform; macOS and Windows compile but are unverified at
> runtime.

---

## What QSL is

QSL is a local-first desktop email client written end-to-end in Rust.
The core is a Tauri 2 + Dioxus 0.7 webview app over a Turso (pure-Rust
SQLite-compatible) store, with first-class support for Gmail
(IMAP + SMTP + OAuth2) and Fastmail (JMAP + OAuth2). The reader is
privacy-first by default: HTML is sanitized through `ammonia`, remote
content is blocked until the sender is trusted, and the message body
renders in a sandboxed iframe with a locked-down CSP. Tokens live in
the OS keychain and are zeroized in memory; no traffic ever traverses
a QSL-controlled server. The chrome is a warm-dark monospace UI in the
aerc/mutt density tradition rather than a webmail clone.

## What's in 0.1

### Accounts & sync

- **Gmail OAuth2 + PKCE** end-to-end, with desktop-app `client_secret`
  flow, libsecret-backed refresh-token storage, and the loopback
  redirect handler in `qsl-auth`.
- **Fastmail JMAP + OAuth2** wired through `qsl-jmap-client` (code
  complete; see Known limitations on live validation).
- **IMAP IDLE** for live push on Gmail with NOOP heartbeats and
  transparent reconnect on stale sessions; **JMAP EventSource** for
  Fastmail.
- **History sync** pulls full mail history per folder with chunked
  FETCH, UID-gap skipping, cancellation that feels instant, and
  per-account serialization. Progress surfaces in the status bar.
- **CONDSTORE / QRESYNC** negotiated; capability set is plumbed
  through to the status bar's protocol indicator.
- **Multi-account** by design — sidebar groups folders per account,
  the unified inbox merges across accounts, and a topbar account
  filter chip scopes the message list.

### Reading mail

- **Sandboxed iframe reader** — `<iframe sandbox="allow-scripts"
  srcdoc="...">` inside webkit2gtk renders the sanitized HTML at a
  unique null origin. No same-origin, no forms, no popups. (The
  embedded Servo renderer that preceded this was removed on
  2026-04-28; see [`docs/servo-tombstone.md`](../servo-tombstone.md)
  for the full story.)
- **Per-sender remote-content opt-in** — images / stylesheets / CSS
  `background-image` URLs are stripped at the sanitizer; a banner
  offers "Load images" (one-shot) and "Always load from this sender"
  (persisted in `remote_content_opt_ins`).
- **Stacked thread reader** — when a message belongs to a thread of
  ≥2, the reader shows every message as a card; the most recent is
  expanded by default, others toggle on click.
- **Popup reader windows** — double-click a message to open it in
  its own Tauri window; `RenderedMessage` is reused from the warm
  cache so first paint is fast.
- **Click forwarder** — anchor clicks postMessage out of the iframe;
  the parent validates `http` / `https` / `mailto` and shells out via
  `webbrowser::open`.

### Writing mail

- **Compose** with reply, reply-all, and forward, seeded with the
  correct `In-Reply-To` / `References` headers.
- **Per-identity signatures** edited from Settings → Accounts.
- **Attachments** via the OS file picker; multipart assembly in
  `qsl-mime` (RFC 5322).
- **Undo-send** with a configurable grace window; the queued send
  sits in the outbox until the timer expires.
- **`mailto:` deep links** — the desktop app registers as a system
  mailto handler and opens compose pre-filled when the user clicks
  a `mailto:` link in another app.
- **Cc / Bcc reveal** — collapsed by default, click to reveal.
- **Address autocomplete** — driven by a write-only `contacts_v1`
  table seeded from every `From:` rendered and every `To:` / `Cc:` /
  `Bcc:` sent.
- **Spell check** via system hunspell (Linux).
- **Right-click context menu** in the compose body (cut / copy /
  paste / paste-as-plain).
- **Drafts** — auto-saved every 5s, synced upstream via the outbox
  (`OP_SAVE_DRAFT` destroys the prior server-side copy on every
  save so the IMAP / JMAP draft folder stays in lockstep with local).

### Lists & search

- **Unified inbox** across every configured account.
- **Full-text search** backed by Turso's experimental
  Tantivy-based FTS index (not SQLite FTS5 — see
  `crates/search/`).
- **Gmail-style operators** — `from:`, `to:`, `subject:`,
  `has:attachment`, `before:`, `after:`, `is:unread`, `in:label`
  parsed into FTS MATCH expressions plus structured filters.
- **Command palette** (`Ctrl+K` / `⌘K`) with live full-text search,
  jump-to-folder, jump-to-account, and recent searches.
- **Thread-count badges** in the message list.
- **Right-click context menus** on message rows for archive / delete /
  flag / mark / move / label.
- **Multi-select with bulk apply** — checkboxes appear on row hover;
  shift-click extends a range; the toolbar shifts to bulk mode and
  applies archive / delete / mark across the selection.
- **Drag-and-drop** messages into folders (move semantics).
- **Keyboard shortcuts** in the Gmail family (`j` / `k`, `e`, `#`,
  `r` / `a` / `f`, `c`, `Esc`, `?` cheatsheet); modifier-prefix
  shortcuts (`⌘K`) bypass the typing-guard so they fire from inside
  inputs.

### Desktop integration

- **System tray** with a per-account unread tooltip; left-click
  raises the main window, right-click exposes Quit / Show.
- **Launch on login** toggle (Settings → Appearance) backed by
  `tauri-plugin-autostart`.
- **Window state restore** (size + maximized + fullscreen only;
  position is not restored on Wayland because the compositor doesn't
  expose absolute coords — see notes in `tauri-plugin-window-state`).
- **Single-instance** enforcement — second invocations raise the
  existing window instead of starting a duplicate.
- **Default-mailto-handler** toggle — register / deregister with the
  desktop environment's URL handler list at runtime.
- **Settings window** (Tauri popout) with Accounts / Appearance /
  Notifications / Shortcuts / Privacy tabs.
- **First-run OAuth** in-app — empty state in the main window opens
  the `oauth-add` Tauri window, runs the loopback flow, and kicks
  the first sync.

### Privacy & security

- **Remote content blocked by default** — `<img src>`, `srcset`,
  `poster`, `background`, inline CSS `background-image` URLs, and
  `<link rel="stylesheet">` are stripped or held back at the
  sanitizer until the sender is trusted.
- **Tracker URL filter** via `adblock-rust` against Brave's curated
  list.
- **Redirect-service unwrap** — outbound clicks have known redirect
  shells (Google, Outlook, Yahoo, etc.) unwrapped to the real
  destination before opening.
- **CSP-locked iframe reader** — the iframe carries its own CSP that
  restricts the rendered email body's egress (a regression test in
  `tests/reader_csp.rs` guards the directive set).
- **Token storage** — refresh tokens live in libsecret /
  Keychain / Credential Manager under service `com.qsl.app`.
  `AccessToken` and `RefreshToken` zeroize on drop. Account removal
  revokes the refresh token at the provider's revocation endpoint
  before clearing the keychain entry.
- **OAuth2 + PKCE** with explicit `state` validation on the loopback
  redirect.

## Known limitations

- **Tauri host-level CSP is not set.** `tauri.conf.json` ships with
  `csp: null`. The iframe reader's own CSP still walls off email
  HTML, but the host webview that frames the Dioxus UI isn't itself
  locked down. PR #131 attempted host-level CSP and was reverted
  after breaking IPC; remediation is the top v0.1 release blocker.
- **Fastmail JMAP path has not been validated against a live
  account.** All `qsl-jmap-client` code (auth, sync, push, send) is
  in place and unit-tested, but no end-to-end smoke against a real
  Fastmail account has been recorded.
- **macOS / Windows builds compile but have not been runtime-tested.**
  Codebase compiles cleanly on all three targets in CI; only Linux
  has been exercised interactively.
- **Gmail OAuth uses the legacy full-mail scope**
  (`https://mail.google.com/`). Narrowing to per-purpose scopes is
  a v0.2 task — the full scope is required by the current IMAP /
  SMTP integration.
- **GTK3-era webkit2gtk dependencies.** Tauri 2 hasn't moved to
  GTK 4 yet, so the gtk-rs ecosystem we depend on (gdk3, gtk3,
  webkit2gtk-4.1) shows up as `unmaintained` in
  `cargo deny check advisories` (~19 crates). Tracked but not
  actionable until upstream Tauri migrates.
- **Reader-pane resize flicker** during continuous window-drags is
  gone (it was a Servo-era issue); the iframe path simply reflows.
  Listed for completeness.
- **No Servo-era features** — process-isolated rendering, GPU
  compositing, and the `qsl-renderer` crate are gone.
  See [`docs/servo-tombstone.md`](../servo-tombstone.md).

The full backlog of consciously-accepted gaps lives in
[`docs/QSL_BACKLOG_FIXES.md`](../QSL_BACKLOG_FIXES.md). The 2026-05-01
security audit is at
[`docs/security/audit-2026-05-01.md`](../security/audit-2026-05-01.md).

## Platforms

- **Linux** — primary platform. webkit2gtk-4.1 with the DMA-BUF
  renderer disabled (`WEBKIT_DISABLE_DMABUF_RENDERER=1`); this is
  baked into `apps/desktop/src-tauri/src/main.rs` so hybrid
  AMD / NVIDIA boxes don't paint blank. hunspell dictionaries
  (`hunspell-en_us`, etc.) are needed for spell-check; QSL will
  start without them but the compose pane won't underline misspellings.
- **macOS** — compiles. Runtime-validation is a v0.1 blocker.
- **Windows** — compiles. Runtime-validation is a v0.1 blocker.

System requirements and Linux package list are in
[`README.md`](../../README.md#build).

## Local data

| Platform | Path |
|---|---|
| **Linux** | `~/.local/share/qsl/` (Turso DB + `blobs/` + outbox) |
| **macOS** | `~/Library/Application Support/app.qsl.qsl/` |
| **Windows** | `%APPDATA%\qsl\qsl\data\` |

Refresh tokens live in the OS keychain under service `com.qsl.app`,
separate from the cache. Maintenance:

- `mailcli reset` — wipe local state.
- `mailcli doctor [--fix --reset-history-sync --rebuild-fts --vacuum --yes]` —
  diagnose and (with `--fix`) repair common database / index drift.
